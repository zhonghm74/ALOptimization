# ALM 90天优化问题描述（parse-lp-input 输出）

## 1. 业务背景
该问题是一个资产负债管理（ALM）线性规划优化任务。机构需要在 90 天滚动周期内，动态配置 15 类资产与 8 类负债，在满足监管约束与经营约束前提下，提高组合整体收益表现并控制风险。

## 2. 优化目标
优化目标为最大化 90 天累计净收益：
- 每日资产收益项：`SUM(Ai, (r[t,Ai]/365) * x[t,Ai])`
- 每日负债成本项：`SUM(Lj, (c[t,Lj]/365) * y[t,Lj])`
- 约束偏离惩罚项：`penalty_s_liq * s_liq[t]` 与 `penalty_s_dur * s_dur[t]`

总体目标可表述为：在 `t=1..90` 上累计上述每日净收益并最大化。

## 3. 决策变量与参数
- 决策变量：
  - `x[t,Ai]`：第 `t` 天资产 `Ai` 持有量
  - `y[t,Lj]`：第 `t` 天负债 `Lj` 融资量
- 松弛变量：
  - `s_liq[t]`：流动性约束松弛
  - `s_dur[t]`：久期约束松弛
- 输入参数：
  - 日度利率参数：`r[t,Ai]`、`c[t,Lj]`
  - 结构参数：`w[Ai]`、`d[Ai]`
  - 跨日调仓上限：`delta_x[Ai]`、`delta_y[Lj]`
  - 策略参数：总资产目标、流动性阈值、久期上限、批发融资上限等

## 4. 关键约束结构
- 每日约束（`t=1..90`）：
  1. 总资产目标约束
  2. 资产负债平衡约束
  3. 流动性覆盖约束（软约束）
  4. 久期暴露约束（软约束）
  5. 批发融资上限约束
  6. 高波动资产组合上限约束
  7. 单一资产集中度约束（A15）
  8. 长期资产与稳定负债匹配约束
  9. 零售存款占比约束
- 跨日约束（`t=2..90`）：
  10. 跨日持仓变动约束（逐标的不同阈值）

## 5. 输出说明
本次 `parse-lp-input` 输出包含：
1. `alm_lp_full_test_input_parsed.json`：结构化 LP 模型（model_json）
2. `alm_lp_full_test_input_variable_name_map_cn.json`：全部变量中文名映射
3. `alm_lp_full_test_input_problem_description.md`：本问题文字说明文档
