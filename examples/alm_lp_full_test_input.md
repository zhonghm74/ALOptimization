## 1.1 问题描述

本案例描述一个 90 天滚动期的资产负债配置线性规划问题。目标是在满足监管和经营约束的前提下，基于每日资产收益率与负债成本率，优化每日资产与负债持仓结构，使 90 天累计净收益最大化。

核心建模要点如下：
- **优化对象**：15 类资产与 8 类负债在 `t=1..90` 每天的持仓规模。
- **目标函数**：按天累计 `资产收益 - 负债成本 - 约束松弛惩罚`。
- **主要约束**：每日总资产目标、资产负债平衡、流动性覆盖、久期暴露、集中度与资金结构约束。
- **动态约束**：针对每个标的设置差异化的跨日持仓变动上限，限制相邻两天仓位跳变。
- **输出期望**：得到可被求解器直接消费的 LP 结构（变量、约束、目标、参数），并可进一步生成求解结果与可视化报告。

| 变量名 | 含义 | 通俗解释 |

## 4. 决策变量范围

- 对每个 `t=1..90` 和每个资产 `Ai`：  
  `lb_Ai <= x[t,Ai] <= ub_Ai`，其中 `lb_Ai` / `ub_Ai` 来自“2.1 资产池”。
- 对每个 `t=1..90` 和每个负债 `Lj`：  
  `lb_Lj <= y[t,Lj] <= ub_Lj`，其中 `lb_Lj` / `ub_Lj` 来自“2.2 负债池”。
- 对每个 `t=1..90`：  
  `s_liq[t] >= 0`，`s_dur[t] >= 0`，两者无显式上界。

## 5. 约束条件（线性）及文字解释

以下约束中：
- 第 1~9 条为“每日存量约束”，对每个 `t=1..90` 都要满足。
- 第 10 条为“跨日变动约束”，对每个 `t=2..90` 都要满足。

1. **总资产目标约束（等式）**  
   ```text
   SUM(i=1..15, x[t,Ai]) = 1200,   for each t=1..90
   ```  
   解释：每天的总资产规模固定为 1200，保证每日配置都落在经营计划规模内。

2. **资产负债平衡约束（等式）**  
   ```text
   SUM(i=1..15, x[t,Ai]) - SUM(j=1..8, y[t,Lj]) = 0,   for each t=1..90
   ```  
   解释：每天资产投放都必须由负债端资金覆盖，确保每日资金来源与运用匹配。

3. **流动性覆盖约束（软约束）**  
   ```text
   SUM(i=1..15, w[Ai] * x[t,Ai]) + s_liq[t] >= 620,   for each t=1..90
   ```  
   解释：按流动性权重折算后的当日流动性资产必须达到 620；若不足，通过 `s_liq[t]` 记录缺口并在目标函数中惩罚。

4. **久期暴露约束（软约束）**  
   ```text
   SUM(i=1..15, d[Ai] * x[t,Ai]) - s_dur[t] <= 4300,   for each t=1..90
   ```  
   解释：当日资产组合久期暴露不应超过 4300；若超出，通过 `s_dur[t]` 表示偏离并在目标函数中惩罚。

5. **批发融资上限约束**  
   ```text
   y[t,L06] + y[t,L07] + y[t,L08] <= 260,   for each t=1..90
   ```  
   解释：每天限制对同业拆入、债券融资及其他高波动融资来源的依赖，控制再融资风险。

6. **高波动资产组合上限约束**  
   ```text
   x[t,A10] + x[t,A11] + x[t,A12] + x[t,A14] + x[t,A15] <= 420,   for each t=1..90
   ```  
   解释：每天约束高波动/高资本占用资产总敞口，防止风险资产过度集中。

7. **单一资产集中度约束（A15）**  
   ```text
   x[t,A15] <= 90,   for each t=1..90
   ```  
   解释：每天对另类固收资产单列集中度上限，避免单一资产类别冲击组合稳健性。

8. **长期资产与稳定负债匹配约束（期限结构代理）**  
   ```text
   x[t,A09] + x[t,A10] + x[t,A11] + x[t,A12]
   <= y[t,L01] + y[t,L02] + y[t,L03] + y[t,L04] + 120,   for each t=1..90
   ```  
   解释：每天要求中长期资产由更稳定的存款负债支持，减少期限错配风险。

9. **零售存款占比约束**  
   ```text
   y[t,L01] + y[t,L02] >= 0.45 * SUM(j=1..8, y[t,Lj]),   for each t=1..90
   ```  
   解释：每天保证负债结构中零售来源占比不低于 45%，提升资金稳定性。

10. **跨日持仓变动约束（逐标的，不同阈值）**  
   ```text
   -delta_x[Ai] <= x[t,Ai] - x[t-1,Ai] <= delta_x[Ai],   for each Ai in A01..A15, t=2..90
   -delta_y[Lj] <= y[t,Lj] - y[t-1,Lj] <= delta_y[Lj],   for each Lj in L01..L08, t=2..90
   ```
   解释：每个标的都设置了各自不同的“单日最大调仓幅度”，限制相邻两天仓位跳变，降低交易冲击、流动性压力和执行风险。

## 6. 优化目标

**目标方向：最大化**

```text
MAX:
  SUM(t=1..90,
    SUM(i=1..15, (r[t,Ai] / 365) * x[t,Ai])
    - SUM(j=1..8, (c[t,Lj] / 365) * y[t,Lj])
    - 0.40 * s_liq[t]
    - 0.25 * s_dur[t]
  )
```

其中：
- `r[t,Ai]`：第 `t` 天资产 `Ai` 的年化收益率（见 2.4）
- `c[t,Lj]`：第 `t` 天负债 `Lj` 的年化资金成本率（见 2.4）
- `x[t,Ai]`、`y[t,Lj]`：第 `t` 天的资产/负债持有量（决策变量）
- `s_liq[t]`、`s_dur[t]`：第 `t` 天约束偏离惩罚项对应松弛变量
- 由于 2.4 提供的是年化率，目标函数按日累计时采用 `年化率 / 365` 转换为单日收益/成本率。

## 7. 结果报告撰写指引（最终交付）

本章节用于规范优化完成后的结果报告输出格式，确保报告既能被管理层快速阅读，也能被模型/分析师复核。

### 7.1 报告目标与读者
- **目标**：说明 90 天优化方案是否实现“收益提升 + 风险可控 + 约束合规”。
- **主要读者**：经营管理层、资金与风险团队、模型审阅人员。
- **输出形式**：`report.md`（文字+图表）与 `report.json`（结构化指标）。

### 7.2 报告建议结构
1. **执行摘要（1页内）**  
   - 目标函数值、约束满足情况、关键资产负债调整结论。
2. **模型与口径说明**  
   - 变量定义、收益口径、成本口径、松弛变量解释。
3. **图表展示（核心）**  
   - 资产份额变化图、累计收益变化图、每日收益变化图。
4. **结果分析与方案解读**  
   - 收益来源拆解、风险暴露变化、稳健性评价。
5. **结论与建议**  
   - 可执行建议、监控指标、后续迭代方向。

### 7.3 必备图表与展示要求

#### 图表A：90天各类资产份额变化图（必须）
- **图意**：展示 15 类资产在 90 天内的配置占比演变，识别结构迁移和集中度变化。
- **建议图型**：堆叠面积图（主图） + 关键资产折线图（辅图）。
- **计算口径**：
  - `asset_share[t,Ai] = x[t,Ai] / SUM(k=1..15, x[t,Ak])`
- **报告中需说明**：
  - 哪些资产份额上升/下降最明显；
  - 是否与收益率变化、流动性权重、久期约束一致；
  - 是否出现集中度风险抬升。

#### 图表B：总收益（累计净收益）变化图（必须）
- **图意**：展示从 D01 到 D90 的累计净收益路径，体现优化方案整体增益趋势。
- **建议图型**：折线图（可附对比基线方案）。
- **计算口径**：
  - `daily_net_pnl[t] = SUM(i, r[t,Ai]/365 * x[t,Ai]) - SUM(j, c[t,Lj]/365 * y[t,Lj]) - 0.40*s_liq[t] - 0.25*s_dur[t]`
  - `cum_net_pnl[t] = SUM(d=1..t, daily_net_pnl[d])`
- **报告中需说明**：
  - 累计收益斜率变化区间；
  - 收益拐点对应的配置变化原因；
  - 与约束紧绑定时段的关系。

#### 图表C：每日收益变化图（必须）
- **图意**：展示每日净收益波动，评估收益稳定性和短期波动风险。
- **建议图型**：柱状图（每日净收益）+ 7日滚动均值折线。
- **计算口径**：
  - `daily_net_pnl[t]` 同上
  - `rolling_7d[t] = AVG(daily_net_pnl[t-6..t])`（`t>=7`）
- **报告中需说明**：
  - 波动高峰与低谷日期；
  - 波动来源（资产端、负债端、松弛惩罚）；
  - 是否存在连续负收益区间及其业务解释。

### 7.4 整体优化方案分析与介绍（必须）
报告正文需至少覆盖以下分析维度：
- **收益维度**：总收益、日均收益、收益波动（标准差/极差）。
- **结构维度**：资产与负债结构迁移、关键品类权重变化。
- **约束维度**：硬约束是否全部满足、软约束触发频率与代价。
- **风险维度**：流动性覆盖、久期暴露、集中度、批发融资依赖度。
- **执行维度**：跨日持仓变动是否频繁触及 `delta_x/delta_y` 上限，方案是否具备落地可执行性。
