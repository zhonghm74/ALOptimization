# 报告要求（自动抽取）

本章节用于规范优化完成后的结果报告输出格式，确保报告既能被管理层快速阅读，也能被模型/分析师复核。

### 7.1 报告目标与读者
- **目标**：说明 90 天优化方案是否实现“收益提升 + 风险可控 + 约束合规”。
- **主要读者**：经营管理层、资金与风险团队、模型审阅人员。
- **输出形式**：`report.md`（文字+图表）与 `report.json`（结构化指标）。

### 7.2 报告建议结构
1. **执行摘要（1页内）**  
   - 目标函数值、约束满足情况、关键资产负债调整结论。
2. **模型与口径说明**  
   - 变量定义、收益口径、成本口径、松弛变量解释。
3. **图表展示（核心）**  
   - 资产份额变化图、累计收益变化图、每日收益变化图。
4. **结果分析与方案解读**  
   - 收益来源拆解、风险暴露变化、稳健性评价。
5. **结论与建议**  
   - 可执行建议、监控指标、后续迭代方向。

### 7.3 必备图表与展示要求

#### 图表A：90天各类资产份额变化图（必须）
- **图意**：展示 15 类资产在 90 天内的配置占比演变，识别结构迁移和集中度变化。
- **建议图型**：堆叠面积图（主图） + 关键资产折线图（辅图）。
- **计算口径**：
  - `asset_share[t,Ai] = x[t,Ai] / SUM(k=1..15, x[t,Ak])`
- **报告中需说明**：
  - 哪些资产份额上升/下降最明显；
  - 是否与收益率变化、流动性权重、久期约束一致；
  - 是否出现集中度风险抬升。

#### 图表B：总收益（累计净收益）变化图（必须）
- **图意**：展示从 D01 到 D90 的累计净收益路径，体现优化方案整体增益趋势。
- **建议图型**：折线图（可附对比基线方案）。
- **计算口径**：
  - `daily_net_pnl[t] = SUM(i, r[t,Ai]/365 * x[t,Ai]) - SUM(j, c[t,Lj]/365 * y[t,Lj]) - 0.40*s_liq[t] - 0.25*s_dur[t]`
  - `cum_net_pnl[t] = SUM(d=1..t, daily_net_pnl[d])`
- **报告中需说明**：
  - 累计收益斜率变化区间；
  - 收益拐点对应的配置变化原因；
  - 与约束紧绑定时段的关系。

#### 图表C：每日收益变化图（必须）
- **图意**：展示每日净收益波动，评估收益稳定性和短期波动风险。
- **建议图型**：柱状图（每日净收益）+ 7日滚动均值折线。
- **计算口径**：
  - `daily_net_pnl[t]` 同上
  - `rolling_7d[t] = AVG(daily_net_pnl[t-6..t])`（`t>=7`）
- **报告中需说明**：
  - 波动高峰与低谷日期；
  - 波动来源（资产端、负债端、松弛惩罚）；
  - 是否存在连续负收益区间及其业务解释。

### 7.4 整体优化方案分析与介绍（必须）
报告正文需至少覆盖以下分析维度：
- **收益维度**：总收益、日均收益、收益波动（标准差/极差）。
- **结构维度**：资产与负债结构迁移、关键品类权重变化。
- **约束维度**：硬约束是否全部满足、软约束触发频率与代价。
- **风险维度**：流动性覆盖、久期暴露、集中度、批发融资依赖度。
- **执行维度**：跨日持仓变动是否频繁触及 `delta_x/delta_y` 上限，方案是否具备落地可执行性。

### 7.5 结论写作模板（可直接复用）
可按以下模板生成最终文字结论：

```text
在90天优化窗口内，方案实现累计净收益 <填入数值>，日均净收益 <填入数值>。
硬约束满足率为 <填入百分比>，软约束触发主要集中在 <填入区间/场景>。
资产结构方面，<填入资产类别> 份额提升显著，主要贡献来自 <收益/成本/约束原因>；
负债结构方面，<填入负债类别> 占比变化显示资金稳定性 <提升/下降>。
综合收益、风险与可执行性评估，建议 <继续执行/局部调参后执行> 当前优化方案。
```
